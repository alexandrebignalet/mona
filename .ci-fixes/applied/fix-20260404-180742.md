## Failure Summary

The Deploy workflow fails during Docker image build because `gradle/wrapper/gradle-wrapper.jar` is excluded by `.dockerignore`, causing `./gradlew shadowJar` to fail with `ClassNotFoundException: org.gradle.wrapper.GradleWrapperMain`.

## Error Output

```
#19 0.350 Error: Could not find or load main class org.gradle.wrapper.GradleWrapperMain
#19 0.350 Caused by: java.lang.ClassNotFoundException: org.gradle.wrapper.GradleWrapperMain
```

## Root Cause

The `.dockerignore` file contains the glob `**/*.jar` which excludes all JAR files from the Docker build context, including the essential `gradle/wrapper/gradle-wrapper.jar`. Without this JAR, the `gradlew` script cannot bootstrap Gradle.

## Fix Plan

1. Edit `.dockerignore` to add a negation rule that re-includes the Gradle wrapper JAR after the `**/*.jar` exclusion:
   - Add `!gradle/wrapper/gradle-wrapper.jar` after the `**/*.jar` line

This is the minimal fix. The `**/*.jar` rule is correct for excluding build artifacts, but needs the wrapper JAR exception.
